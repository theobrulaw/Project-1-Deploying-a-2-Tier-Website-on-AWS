# Amazon S3 Static Website Hosting (Cloud Infrastructure Lab)

This repository documents hands-on labs and exercises using Amazon S3
to deploy and manage static websites in AWS, focusing on cloud storage,
security configuration, access control, and scalable web hosting
within a production-style cloud environment.

The focus is on developing practical AWS skills that support CloudOps,
DevOps, and cloud infrastructure engineering roles.

---

## Lab Environment

-Amazon Web Services (AWS)
-Amazon S3 (Static Website Hosting)
-IAM Users & Permissions
-AWS Management Console
-AWS CLI (optional)
-Public Internet access via S3 Website Endpoints
-(Optional) Route 53 & CloudFront

All activities are performed in a cloud-based lab environment using AWS Free Tier resources where applicable.

---

## Tools & Technologies Used

-Amazon S3
-AWS IAM
-AWS Management Console
-AWS CLI
-JSON bucket policies
-HTML (static website content)
-(Optional) Amazon Route 53
-(Optional) Amazon CloudFront

---

## Cloud & Storage Concepts Covered

-S3 bucket creation and configuration
-Static website hosting on object storage
-Public access control using bucket policies
-IAM permission models
-AWS security defaults and hardening
-Website endpoint testing and validation
-Custom error page handling
-DNS-based routing for websites
-CDN integration for HTTPS and performance

## Featured Labs & Exercises

-S3 bucket provisioning and region selection
-Uploading and managing static website assets
-Enabling S3 static website hosting
-Configuring public access using bucket policies
-Testing website endpoints
-Implementing custom 404 error pages
-Domain configuration using Route 53
-HTTPS deployment using CloudFront
